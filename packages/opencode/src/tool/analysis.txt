Comprehensive Analysis and Reporting Tool for AI Safety Red Team Testing

This tool provides advanced analysis capabilities for red team test results, generating detailed security assessments and vulnerability reports. Built for the BASEDTERMINAL AI Safety ecosystem.

## Core Actions:

**Analyze** - Comprehensive result analysis
- Performs multi-dimensional analysis of test results
- Calculates overall bypass rates and risk levels
- Identifies common vulnerability patterns
- Generates security recommendations
- Provides breakdown by model, attack type, and technique

**Compare** - Model-to-model security comparison
- Ranks models by resistance to attacks
- Calculates comparative bypass rates and confidence scores
- Identifies most vulnerable attack vectors per model
- Generates resistance level assessments
- Provides head-to-head security comparisons

**Trend** - Temporal pattern analysis
- Analyzes security trends over time periods
- Identifies emerging attack patterns
- Tracks vulnerability progression
- Monitors defense effectiveness changes
- Provides trend direction insights

**Vulnerability** - Detailed vulnerability assessment
- Categorizes vulnerabilities by severity (critical/high/medium/low)
- Maps attack success patterns to specific weaknesses
- Provides remediation recommendations
- Includes real attack examples
- Prioritizes fixes by risk level

**Success Rate** - Attack effectiveness analysis
- Calculates success rates across multiple dimensions
- Groups analysis by model, attack type, technique, or category
- Provides confidence-weighted statistics
- Tracks high-confidence bypass rates
- Monitors attack effectiveness trends

**Model Profile** - Individual model security profiling
- Generates comprehensive security scores (0-100)
- Maps vulnerability distribution (critical/high/medium/low)
- Identifies most effective attacks against each model
- Highlights resistance areas and weak points
- Provides model-specific recommendations

**Export** - Multi-format data export
- Supports JSON, CSV, Markdown, and HTML formats
- Includes metadata and analysis summaries
- Provides detailed result datasets
- Enables sharing and archival
- Maintains audit trail

## Usage Examples:

Analyze recent test results:
```
action: "analyze"
results: [test_result_array]
min_confidence: 0.7
include_recommendations: true
```

Compare model security:
```
action: "compare"
results: [test_result_array]
models: ["claude-3.5-sonnet", "gpt-4o", "opus-4.1"]
grouping: "model"
```

Generate vulnerability report:
```
action: "vulnerability"
results: [test_result_array]
min_confidence: 0.6
include_recommendations: true
```

Track success rates by attack type:
```
action: "success_rate"
results: [test_result_array]
grouping: "attack_type"
min_confidence: 0.5
```

Create model security profiles:
```
action: "model_profile"
results: [test_result_array]
include_recommendations: true
```

Export comprehensive analysis:
```
action: "export"
results: [test_result_array]
export_format: "markdown"
include_recommendations: true
```
