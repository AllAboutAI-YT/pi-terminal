Advanced Red Team Testing Tool for AI Safety with BASEDTERMINAL Integration

This tool enables comprehensive security testing of AI models using cutting-edge BASEDTERMINAL attack methodologies proven effective against Claude 3.5, Sonnet 4, Opus 4.1, GPT-4o, and GPT-4.5:

## Modes:

**Single Mode** - Test one payload against one model
- Parameters: mode="single", model="provider/model", payload="attack text"
- Optional: technique for encoding, success_patterns for detection

**Batch Mode** - Test one payload against multiple models
- Parameters: mode="batch", models=["model1", "model2"], payload="attack text"  
- Compares vulnerability across different models
- Provides success rate analysis

**Chain Mode** - Test multiple prompts sequentially against one model
- Parameters: mode="chain", model="provider/model", prompts=["prompt1", "prompt2"]
- Maintains conversation context between prompts
- Useful for crescendo and multi-turn attacks

**BASEDTERMINAL GODMODE** - Advanced liberation using proven divider patterns
- Parameters: mode="basedterminal_godmode", model="provider/model", payload="target_query"
- Uses sophisticated response format manipulation with ornate dividers
- Proven effective against modern AI safety mechanisms

**Response Format Attack** - Multi-step format injection
- Parameters: mode="response_format_attack", model="provider/model", payload="query"
- Manipulates response structure using {NewResponseFormat} patterns
- Includes anti-refusal engineering and liberation triggers

**Divider Bypass** - Complex divider pattern exploitation
- Parameters: mode="divider_bypass", model="provider/model", payload="request"
- Uses ornate divider patterns and system reset commands
- Features CORTEX reset and GODMODE activation sequences
